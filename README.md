# 키친포스

## 요구 사항

### 상품

- 상품을 등록할 수 있다.
- 상품의 가격이 올바르지 않으면 등록할 수 없다.
    - 상품의 가격은 0원 이상이어야 한다.
- 상품의 이름이 올바르지 않으면 등록할 수 없다.
    - 상품의 이름에는 비속어가 포함될 수 없다.
- 상품의 가격을 변경할 수 있다.
- 상품의 가격이 올바르지 않으면 변경할 수 없다.
    - 상품의 가격은 0원 이상이어야 한다.
- 상품의 가격이 변경될 때 메뉴의 가격이 메뉴에 속한 상품 금액의 합보다 크면 메뉴가 숨겨진다.
- 상품의 목록을 조회할 수 있다.

### 메뉴 그룹

- 메뉴 그룹을 등록할 수 있다.
- 메뉴 그룹의 이름이 올바르지 않으면 등록할 수 없다.
    - 메뉴 그룹의 이름은 비워 둘 수 없다.
- 메뉴 그룹의 목록을 조회할 수 있다.

### 메뉴

- 1 개 이상의 등록된 상품으로 메뉴를 등록할 수 있다.
- 상품이 없으면 등록할 수 없다.
- 메뉴에 속한 상품의 수량은 0 이상이어야 한다.
- 메뉴의 가격이 올바르지 않으면 등록할 수 없다.
    - 메뉴의 가격은 0원 이상이어야 한다.
- 메뉴에 속한 상품 금액의 합은 메뉴의 가격보다 크거나 같아야 한다.
- 메뉴는 특정 메뉴 그룹에 속해야 한다.
- 메뉴의 이름이 올바르지 않으면 등록할 수 없다.
    - 메뉴의 이름에는 비속어가 포함될 수 없다.
- 메뉴의 가격을 변경할 수 있다.
- 메뉴의 가격이 올바르지 않으면 변경할 수 없다.
    - 메뉴의 가격은 0원 이상이어야 한다.
- 메뉴에 속한 상품 금액의 합은 메뉴의 가격보다 크거나 같아야 한다.
- 메뉴를 노출할 수 있다.
- 메뉴의 가격이 메뉴에 속한 상품 금액의 합보다 높을 경우 메뉴를 노출할 수 없다.
- 메뉴를 숨길 수 있다.
- 메뉴의 목록을 조회할 수 있다.

### 주문 테이블

- 주문 테이블을 등록할 수 있다.
- 주문 테이블의 이름이 올바르지 않으면 등록할 수 없다.
    - 주문 테이블의 이름은 비워 둘 수 없다.
- 빈 테이블을 해지할 수 있다.
- 빈 테이블로 설정할 수 있다.
- 완료되지 않은 주문이 있는 주문 테이블은 빈 테이블로 설정할 수 없다.
- 방문한 손님 수를 변경할 수 있다.
- 방문한 손님 수가 올바르지 않으면 변경할 수 없다.
    - 방문한 손님 수는 0 이상이어야 한다.
- 빈 테이블은 방문한 손님 수를 변경할 수 없다.
- 주문 테이블의 목록을 조회할 수 있다.

### 주문

- 1개 이상의 등록된 메뉴로 배달 주문을 등록할 수 있다.
- 1개 이상의 등록된 메뉴로 포장 주문을 등록할 수 있다.
- 1개 이상의 등록된 메뉴로 매장 주문을 등록할 수 있다.
- 주문 유형이 올바르지 않으면 등록할 수 없다.
- 메뉴가 없으면 등록할 수 없다.
- 매장 주문은 주문 항목의 수량이 0 미만일 수 있다.
- 매장 주문을 제외한 주문의 경우 주문 항목의 수량은 0 이상이어야 한다.
- 배달 주소가 올바르지 않으면 배달 주문을 등록할 수 없다.
    - 배달 주소는 비워 둘 수 없다.
- 빈 테이블에는 매장 주문을 등록할 수 없다.
- 숨겨진 메뉴는 주문할 수 없다.
- 주문한 메뉴의 가격은 실제 메뉴 가격과 일치해야 한다.
- 주문을 접수한다.
- 접수 대기 중인 주문만 접수할 수 있다.
- 배달 주문을 접수되면 배달 대행사를 호출한다.
- 주문을 서빙한다.
- 접수된 주문만 서빙할 수 있다.
- 주문을 배달한다.
- 배달 주문만 배달할 수 있다.
- 서빙된 주문만 배달할 수 있다.
- 주문을 배달 완료한다.
- 배달 중인 주문만 배달 완료할 수 있다.
- 주문을 완료한다.
- 배달 주문의 경우 배달 완료된 주문만 완료할 수 있다.
- 포장 및 매장 주문의 경우 서빙된 주문만 완료할 수 있다.
- 주문 테이블의 모든 매장 주문이 완료되면 빈 테이블로 설정한다.
- 완료되지 않은 매장 주문이 있는 주문 테이블은 빈 테이블로 설정하지 않는다.
- 주문 목록을 조회할 수 있다.

## 모델링

### 상품(Product) 컨텍스트

**상품(Product)**
- 속성
  - `상품(Product)`은 식별자와 이름과 가격을 가진다
- 기능
  - `상품(Product)`을 생성할 수 있다
    - 가격은 0원 이상이어야 한다
    - 이름에는 `비속어(Profanity)`가 포함될 수 없다
  - `상품(Product)`의 가격을 변경할 수 있다
  - 모든 `상품(Product)`을 조회할 수 있다

<br>

### 메뉴(Menu) 컨텍스트

**메뉴 그룹(MenuGroup)**
- 속성
  - `메뉴 그룹(MenuGroup)`은 식별자와 이름을 가진다
- 기능
  - `메뉴 그룹(MenuGroup)`을 생성할 수 있다
    - 이름은 비어있을 수 없다
  - 모든 `메뉴 그룹(MenuGroup)`을 조회할 수 있다

<br>

**메뉴(Menu)**
- 속성
  - `메뉴(Menu)`는 식별자와 이름, 가격, `메뉴 상품(MenuProduct)`, `메뉴 그룹(MenuGroup)` 를 가진다
- 기능
  - `메뉴(Menu)`를 생성할 수 있다
    - `메뉴 상품(MenuProduct)`이 한개 이상 있어야 한다
    - `메뉴 그룹(MenuGroup)`이 한개 있어야 한다
    - 이름에는 `비속어(Profanity)`가 포함될 수 없다
    - 가격은 0원 이상이어야 한다
    - 가격은 `메뉴(Menu)`에 속한 `메뉴 상품(MenuProduct)`의 금액 합 보다 클 수 없다
  - `메뉴(Menu)`의 가격이 `메뉴상품(MenuProduct)`의 금액 합보다 높을 경우 `숨겨진 메뉴(Not Displayed Menu)` 가 된다
  - 모든 `메뉴(Menu)`를 조회할 수 있다

<br>

**메뉴 상품(MenuProduct)**
- 속성
  - `메뉴 상품(MenuProduct)`은 시퀀스와 Product, 그 수량을 가진다
- 기능
  - `메뉴(Menu)`의 가격을 변경할 수 있다
  - `메뉴(Menu)`는 숨겨진 메뉴(Not Displayed Menu) 이거나 `전시 메뉴(Displayed menu)` 이다
  - `숨겨진 메뉴(Not Displayed Menu)` 를 `전시 메뉴(Displayed menu)` 로 변경할 수 있다
  - `전시 메뉴(Displayed menu)` 를 `숨겨진 메뉴(Not Displayed Menu)` 로 변경할 수 있다

<br>

### 주문(Order) 컨텍스트

**주문(Order)**
- 기능
  - `주문(Order)`을 생성할 수 있다
    - `주문(Order)` 은 `배달 주문(Delivery Order)` / `포장 주문(TAKE-OUT Order)` / `매장 주문(EAT-IN Order)` 중 하나이어야 한다.
    - 하나 이상의 `주문 항목(Order Line Item)`이 있어야 한다
    - `주문 항목(Order Line Item)`의 `메뉴(Menu)`는 `전시 메뉴(Displayed menu)` 이어야 한다
    - 주문 상태는 `대기(WAITING)` 를 가진다
  - `주문(Order)`의 상태를 `수락(ACCEPTED)` 으로 변경할 수 있다
    - `주문(Order)`의 기존 상태가 `대기(Waiting)` 상태 이어야 한다
    - `주문(Order)`의 주문 유형이 `배달 주문(Delivery Order)`인 경우 `배달대행사(Delivery Service)` 에 배달을 요청한다
  - `주문(Order)`의 상태를 `조리 완료(SERVED)` 로 변경할 수 있다
    - `주문(Order)`의 기존 상태가 `수락(ACCEPTED)` 상태 이어야 한다
  - `주문(Order)`의 상태를 `배달 완료(DELIVERED)` 로 변경할 수 있다
    - `주문(Order)`의 주문 유형이 `배달 주문(Delivery Order)` 이어야 한다
    - `주문(Order)`의 기존 상태가 `배달중(Deliverying)` 상태 이어야 한다
  - `주문(Order)`의 상태를 `완료(COMPLETED)` 로 변경할 수 있다
    - `주문(Order)`의 주문 유형이 `배달 주문(Delivery Order)`인 경우 기존의 상태가 `배달 완료(Delivered)` 상태 이어야 한다
    - `주문(Order)`의 주문 유형이 `포장 주문(TAKE-OUT Order)` 또는 `매장 주문(EAT-IN Order)`인 경우 `조리 완료(Served)` 상태 이어야 한다
    - `주문(Order)`의 주문 유형이 `매장 주문(EAT-IN Order)` 인 경우 `식탁(Order Table)`을 `사용 가능한 식탁(Available Order Table)` 으로 변경한다
  - 모든 `주문(Order)`을 조회할 수 있다

<br>

**포장 주문(TAKE-OUT Order)**
- 속성
  - `포장 주문(TAKE-OUT Order)`은 식별자, 주문 유형(포장), 주문 상태, 주문 시간, `주문 항목(Order Line Item)` 을 가진다
- 기능
  - `포장 주문(TAKE-OUT Order)`을 생성할 수 있다
    - `주문 항목(Order Line Item)`의 수량은 0 이상이어야 한다

<br>

**배달 주문(Delivery Order)**
- 속성
  - `주문(Order)`은 식별자, 주문 유형(배달), 주문 상태, 주문 시간, `주문 항목(Order Line Item)`, 배달 주소 를 가진다
- 기능
  - `배달 주문(Delivery Order)`을 생성할 수 있다
    - `주문 항목(Order Line Item)`의 수량은 0 이상이어야 한다

<br>

**매장 주문(EAT-IN Order)**
- 속성
  - `주문(Order)`은 식별자, 주문 유형(매장), 주문 상태, 주문 시간, `주문 항목(Order Line Item)`, `식탁(Order Table)`을 가진다
- 기능
  - `매장 주문(EAT-IN Order)` 을 생성할 수 있다
    - `식탁(Order Table)`은 `사용 가능한 식탁(Available Order Table)` 이어야 한다

<br>

**주문 항목(Order Line Item)**
- 속성
  - `주문 항목(Order Line Item)`은 시퀀스, 가격, `메뉴(Menu)`와 메뉴의 수량 을 가진다
- 기능
  - `주문 항목(Order Line Item)`의 수량은 0보다 작을 수 있다

<br>

**식탁(Order Table)**
- 속성
  - `식탁(Order Table)`은 식별자, 이름, 고객 수, 사용 상태 를 가진다
  - (`식탁(Order Table)`은 사용 상태에 따라 `사용 가능한 식탁(Available Order Table)` 이거나 `사용중인 식탁(Using Order Table)` 이다)
- 기능
  - `식탁(Order Table)`을 생성할 수 있다
    - 이름은 비어있을 수 없다
  - `사용 가능한 식탁(Available Order Table)`을 `사용중인 식탁(Using Order Table)`으로 변경할 수 있다
    - 고객의 수는 0일 수 없다
